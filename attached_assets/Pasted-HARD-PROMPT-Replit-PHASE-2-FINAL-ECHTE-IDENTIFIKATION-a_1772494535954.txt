C) BACKEND — ECHTE IDENTIFIKATION (MUST implement, robust)
Wir ersetzen Mock durch OCR+Matching. Der Identify-Output bleibt gleich.

C1) Endpoints
1) POST /api/whisky/identify        (multipart form-data image OR {photoUrl})
2) POST /api/whisky/identify-text   (JSON { query: string })
Optional:
3) POST /api/whisky/identify-barcode (JSON { ean: string }) — only if quick

C2) OCR Provider Abstraktion (server/lib/ocr.ts)
- Choose best available provider:
  a) If GOOGLE_VISION_API_KEY or credentials exist → use Google Cloud Vision OCR
  b) else if AWS creds exist → Textract
  c) else attempt local OCR only if dependable in this runtime (avoid breaking builds)
  d) else return empty candidates gracefully
No TODOs; implement at least one working provider path; otherwise fail gracefully.

C3) Normalization + Entity Extraction (server/lib/normalize.ts)
From OCR text / user query:
- uppercase, remove punctuation
- remove noise words (SINGLE, MALT, SCOTCH, WHISKY, DISTILLERY, LIMITED, EDITION, THE, SMALL, BATCH)
- extract:
  - abv: (\d{2}\.?\d?)\s*%
  - age: (\d{1,2})\s*(YO|YEARS|YEAR)
  - cask tokens: SHERRY, PX, OLOROSO, BOURBON, PORT, RUM, WINE, MADEIRA, VIRGIN OAK
- build token list + normalized string

C4) Local DB Index (server/lib/whiskyIndex.ts)
- Build an in-memory index of local whiskies:
  whiskyId, name, distillery?, abv?, age?
  searchableString, tokens, trigrams
- Refresh on server start; add simple TTL refresh if needed.
If no whisky catalog exists, return candidates=[] and log warning.

C5) Matching (server/lib/matching.ts)
Compute confidence score 0..1 combining:
- token overlap (Jaccard)
- trigram similarity (Dice coefficient) on normalized strings
- bonuses for age/abv matches
Return top 3 sorted.
If top < 0.55 => candidates=[]

C6) “Menu/Text Photo” special handling
If OCR text seems to include multiple whisky-like items:
- Heuristic: split lines, detect lines with (AGE or ABV or known distillery tokens)
- Generate candidate sets per line
- Aggregate and deduplicate candidates, keep top 6 internally, but return top 3 in API.
- Include debug hint: debug.detectedMode = "menu" | "label" | "text"
Front-end uses this only to show “Multiple items detected”.

C7) Caching (server/lib/cache.ts)
- Hash image bytes SHA256
- LRU cache response for 24h
- Cache key also for identify-text queries (normalized query)

C8) Rate limiting
- Identify endpoints: 5 requests per 5 minutes per IP/session (minimal middleware)
- Must not block normal usage; show clear error message.

C9) External lookup (OPTIONAL ONLY IF CONFIGURED)
If EXTERNAL_WHISKY_LOOKUP_PROVIDER is set and keys exist:
- Use provider to fetch candidates by name/distillery/age
- Merge into candidates with source="external"
If not configured: skip silently.
