========================
PART 1 – METHODICAL SAFEGUARDS (NON-NEGOTIABLE)
========================

1) NO TYPOLOGIES  
Do NOT create personality types or categories.  
No labels like “Explorer” or “Connoisseur”.  
Model must be dimensional only.

2) DIMENSIONAL & DYNAMIC  
Taste is a multidimensional, continuous preference structure.  
Profiles evolve with new data.

3) BEHAVIOR-BASED  
All insights must derive from observed behavior (ratings, selections).  
No psychological traits may be inferred.

4) NO PSYCHOLOGICAL ATTRIBUTIONS  
Never say “You are strict.”  
Instead say:  
“Your rating style shows a stable positive deviation from the platform median.”

5) DESCRIPTIVE, NOT NORMATIVE  
No better/worse.  
No percentile ranks.  
No leaderboard framing in profile context.

6) IPSATIVE FIRST  
Default view must show only self-based analysis.  
Comparison must be activated manually.

7) FULL TRANSPARENCY  
All comparisons must show:
- Sample size (N)
- Median basis
- Dispersion (IQR if possible)
- Stability level
Include “Details anzeigen” collapsible panels.

8) ROBUST STATISTICS  
Use median for platform baseline.  
Use IQR if available.  
Avoid mean where sensitive to outliers.

9) CONFIDENCE / STABILITY REQUIRED  
All dimensions must show stability based on:
- number of observations
- variance
- data coverage

10) PLATFORM BASIS = ALL USERS  
Include all active profiles.  
Display sample size clearly.

11) NO OVERINTERPRETATION  
Describe deviations structurally, never psychologically.

12) DYNAMIC SYSTEM  
Platform metrics and user profiles evolve over time.  
Make this explicit in method page.

If any feature risks violating these rules, pause and ask for clarification.
