# CaskSense — Navigation Redesign
## Standard: Apple Human Interface Guidelines · WCAG AA · Senior-Level

---

## Kontext & Priorität

Echtes Nutzerfeedback (Udo, Ersttester):
- Solo-Dram-Funktion bricht beim ersten Versuch
- Einstieg zu verwirrend, zu viele Optionen gleichzeitig
- Text auf kleinem Screen ohne Brille zu blass
- Frage: Was ist der primäre Zweck der App?

Diese drei Probleme lösen wir in dieser Reihenfolge:
1. Bug: Solo-Dram-Identifikation schlägt fehler nach Foto
2. Navigation: 5 Tabs → 4 Tabs, klare Hierarchie
3. Kontrast: Alle Texte auf WCAG AA anheben

---

## TEIL 1 — Bug fix: Solo-Dram Foto-Identifikation

### Problem
Nach Kamera-Aufnahme in der Solo-Capture-Flow schlägt die KI-Identifikation
mit einem generischen Fehler fehl. Der Nutzer landet in einem Fehlerzustand
ohne klaren Weg zurück.

### Fix in `client/src/labs/pages/LabsTastings.tsx` (Solo-Flow)

Stelle sicher dass der catch-Block im Solo-Capture-Flow immer einen
nutzerfreundlichen Zustand rendert:

```tsx
// Suche den try/catch rund um die Identifikations-API
// VORHER: generischer catch ohne UX
catch (error) {
  console.error(error);
  // ... kein UI-Feedback
}

// NACHHER: immer ein klarer Fehlerzustand mit Ausweg
catch (error) {
  console.error('[Solo Capture] Identification failed:', error);
  setSoloView('editor'); // zurück zum Editor
  setIdentificationError(
    'Die Flasche konnte nicht erkannt werden. ' +
    'Du kannst den Whisky manuell eingeben.'
  );
}
```

Füge direkt unter dem Foto-Bereich im Editor-State eine Fehleranzeige ein:

```tsx
{identificationError && (
  <div style={{
    padding: '12px 16px',
    borderRadius: '10px',
    border: '0.5px solid currentColor',
    opacity: 0.7,
    fontSize: '13px',
    marginBottom: '1rem',
    fontFamily: 'var(--font-ui)',
    lineHeight: 1.5,
  }}>
    {identificationError}
  </div>
)}
```

## TEIL 2 — Navigation: 5 Tabs → 4 Tabs

### Datei: `client/src/labs/LabsLayout.tsx`

### Neue Tab-Struktur

```
HEUTE:   Tastings | Explore | Discover | Taste | Circle
NEU:     Tastings | Entdecken | Meine Welt | Circle
```

### Warum genau diese 4

- **Tastings** — Der primäre Job. Alles Aktive: joinen, hosten, solo loggen.
- **Entdecken** — Explore (Whiskies) + Discover (Wissen) unter einem Dach.
  Der Nutzer denkt nicht in dieser Unterscheidung.
- **Meine Welt** — Taste, Analytics, Connoisseur, Journal, Compare, Collection.
  Alles Persönliche an einem Ort.
- **Circle** — Soziales. Kommt bewusst zuletzt — erst relevant nach echten Tastings.

## TEIL 4 — Kontrast: WCAG AA für alle Texte

### Udo's Feedback: „Schrift sehr blässlich ohne Brille auf kleinem Screen"

Apple verlangt WCAG AA: mindestens 4.5:1 für normalen Text, 3:1 für großen Text.

### In `labs-theme.css` — Opacity-Werte anheben:

```css
/* VORHER — zu blasse Werte */
--labs-text-muted: rgba(255, 255, 255, 0.4);
--labs-text-secondary: rgba(255, 255, 255, 0.55);
--labs-border: rgba(255, 255, 255, 0.08);

/* NACHHER — WCAG AA konform */
--labs-text-muted:     rgba(255, 255, 255, 0.55);  /* war 0.4 */
--labs-text-secondary: rgba(255, 255, 255, 0.72);  /* war 0.55 */
--labs-text-primary:   rgba(255, 255, 255, 0.96);  /* war 0.9 */
--labs-border:         rgba(255, 255, 255, 0.14);  /* war 0.08 */
```

### Zusätzlich — minimale Schriftgröße erhöhen:

```css
/* In labs-theme.css */
/* VORHER */
.ty-caption { font-size: 11px; }
.ty-label   { font-size: 10px; }

/* NACHHER — lesbarer auf kleinen Screens */
.ty-caption { font-size: 12px; }
.ty-label   { font-size: 11px; letter-spacing: 0.11em; }
```

### Icon-Strich stärker:

```css
/* Alle Nav-Icons: strokeWidth von 1 auf 1.5 erhöhen im inaktiven Zustand */
/* Aktiver Zustand: 2.0 */
/* Kein Icon darf dünner als 1.5px sein */
```

## Reihenfolge der Umsetzung

1. **Zuerst**: Bug-Fix Solo-Dram (TEIL 1) — kein Nutzer darf beim ersten Klick scheitern
2. **Dann**: Kontrast-Fix (TEIL 4) — sofortiger sichtbarer Unterschied für alle
3. **Dann**: Nav-Struktur (TEIL 2) — 5 → 4 Tabs
4. **Dann**: Entdecken-Hub (TEIL 3) — neue Seite als Eingangstür
5. **Zuletzt**: Routen (TEIL 5) — alles verknüpfen

---

## Was NICHT geändert wird

- Keine Backend-Änderungen
- Keine Datenbankänderungen
- Keine bestehenden Seiten-Inhalte (nur die Nav und die neue Hub-Seite)
- Alle Deep Links bleiben funktionsfähig
- Routing-Logik von `SmartRedirectToLabs` bleibt unberührt
- i18n-Keys bleiben erhalten — nur neue Keys werden ergänzt

---

## Das Designprinzip dahinter (für Code-Reviews)

Navigation soll verschwinden, nicht auffallen.
Ein Nutzer der über die Navigation nachdenkt hat schon verloren.
Vier Tabs, klare Labels, kein Hintergrund-Chip, keine Balken.
Nur Farbe und Fontgewicht zeigen den aktiven Zustand.
Das ist Apple. Das reicht.
