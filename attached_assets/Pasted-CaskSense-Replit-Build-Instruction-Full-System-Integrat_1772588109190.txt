CaskSense – Replit Build Instruction (Full System Integration)

Goal:
Analyze the existing CaskSense codebase and extend it to support
hosted tastings, participant ratings and analytics-driven taste insights.

Important:
Do NOT overwrite existing working features.
First inspect the current project structure and reuse existing logic where possible.

--------------------------------
1. Analyze Current Codebase
--------------------------------

Inspect:

client/
server/
db/
routes/
schema.ts
existing API endpoints

Identify existing features for:

Join a Tasting
Log a Whisky
My Taste

Reuse existing components and database tables whenever possible.

Only extend missing functionality.

6. Analytics Layer
--------------------------------

Analytics must live inside:

My Taste → Analytics

Analytics unlock after:

10 logged whiskies.

Before threshold show progress.

--------------------------------

Analytics modules:

Taste Profile
Taste Map
Taste Evolution
Rating Consistency
Taste Identity

--------------------------------
7. Taste Identity Engine
--------------------------------

Create backend service:

/server/analytics/tasteEngine.ts

Compute analytics using ratings table.

Modules:

Taste Profile
Attribute frequency aggregation.

Taste Map
Flavor distribution.

Taste Evolution
Score trends by month.

Rating Consistency
Standard deviation of ratings.

Taste Identity
Rule-based classification.

Example labels:

Peat Explorer
Sherry Lover
Balanced Taster
Fruit Driven
Spice Enthusiast
