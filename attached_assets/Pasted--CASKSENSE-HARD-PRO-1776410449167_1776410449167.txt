SCHRITT 2 — HÖCHSTE / NIEDRIGSTE: LEERE KACHELN FIXEN (Befund #27)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Das Problem: highestWhisky und lowestWhisky sind leer weil der
dataSnapshot beim Generieren keine Draft-Einträge eingeschlossen hat.

OPTION A — Live-Query (bevorzugt, wenn einfach umsetzbar):
Statt aus dataSnapshot zu lesen: lade highestWhisky und lowestWhisky
direkt live aus journal_entries für diesen Teilnehmer.

In server/routes.ts, im GET-Endpoint für connoisseur-report:
Füge nach dem Laden des Reports eine zusätzliche Query hinzu:

// Höchster bewerteter Whisky (inkl. Drafts mit Score)
const highestEntry = await db
.select()
.from(journalEntries)
.where(
and(
eq(journalEntries.participantId, participantId),
isNotNull(journalEntries.overallScore)
)
)
.orderBy(desc(journalEntries.overallScore))
.limit(1);

// Niedrigster bewerteter Whisky (inkl. Drafts mit Score)
const lowestEntry = await db
.select()
.from(journalEntries)
.where(
and(
eq(journalEntries.participantId, participantId),
isNotNull(journalEntries.overallScore)
)
)
.orderBy(asc(journalEntries.overallScore))
.limit(1);

Ergänze diese Werte in der API-Response neben den dataSnapshot-Daten:
highestWhisky: highestEntry[0] ?? null,
lowestWhisky: lowestEntry[0] ?? null,

OPTION B — Fallback im Frontend (falls Option A zu komplex):
In der Connoisseur-Seite: wenn highestWhisky null ist, zeige statt “—”
den Text:
DE: “Bericht neu erstellen um anzuzeigen”
EN: “Regenerate report to display”
in 11px muted color — kein “—” mehr.

WICHTIG: Implementiere NUR eine der beiden Optionen.
Wähle Option A wenn der Aufwand überschaubar ist, sonst Option B.
Berichte welche Option umgesetzt wurde.
