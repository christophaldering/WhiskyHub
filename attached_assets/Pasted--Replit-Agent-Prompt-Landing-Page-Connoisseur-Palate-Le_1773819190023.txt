# TEIL 2: CONNOISSEUR “PALATE LETTER”

## Philosophie

Der Connoisseur Report ist kein Feature das man aufruft.
Er ist ein Moment der den Nutzer findet.

Methodischer Anspruch: revealed preference statt Selbstbeschreibung.
Drei Ebenen: deskriptiv → interpretativ (vorsichtig) → reflexiv (offene Frage).
Ton: warm, präzise, epistemisch bescheiden. Kein Persönlichkeitsprofil.
Sprache: immer in der vom Nutzer gewählten Sprache (participant.language oder i18n.language).

-----

## AKT 1: STILLE BEOBACHTUNG

In allen Labs-Pages: Entferne Hinweise die den Nutzer auf den Report hinweisen
bevor er bereit ist. Konkret:

- “X / 10 Drams bis zu deinem Connoisseur Report” Texte
- Fortschrittsbalken in Richtung Connoisseur Report
- Badges oder Counter die auf den Report hinweisen

Der bestehende locked-state Progress in `LabsTasteAnalytics.tsx` bleibt —

### Trigger-Logik

In `LabsLayout.tsx`, beim Mount:

```typescript
const checkForPaletteLetterTrigger = async () => {
  const alreadySeen = localStorage.getItem('casksense_letter_seen');
  if (alreadySeen) return;
  
  // Ratings-Count aus bestehendem API laden
  // z.B. GET /api/participants/:id/stats oder flavor profile endpoint
  const ratingsCount = await fetchRatingsCount(participantId);
  
  if (ratingsCount >= 10) {
    // Flag noch nicht setzen — erst nach dem der Nutzer den Screen gesehen hat
    setShowPaletteLetter(true);

};
```

## AKT 3: DER REPORT SELBST

### GPT-4o Prompt überarbeiten

Endpoint: `POST /api/participants/:id/connoisseur-report`
Datei: `server/routes.ts` oder dediziertes connoisseur-routes File.

### Datenbasis erweitern

Zusätzlich zu den bestehenden Daten (ratings, journal, collection, flavor indices)
folgende Signale berechnen und in den Prompt einspeisen:

```typescript
const analyticsData = {
  // Bestehend:
  ratings: allRatings,
  journalEntries: journalEntries,
  flavorProfile: flavorProfileData,
  
  // NEU — vor dem API Call berechnen:
  
  // 1. Abweichung vom Community-Schnitt pro Dimension
  dimensionDeviations: {
    nose: userAvgNose - communityAvgNose,
    palate: userAvgPalate - communityAvgPalate,
    finish: userAvgFinish - communityAvgFinish,
    balance: userAvgBalance - communityAvgBalance,
  },
  
  // 2. Systematische Stil-Präferenzen (revealed, nicht deklariert)
  styleDeviations: {
    // Differenz user avg vs community avg, gruppiert nach:
    byRegion: groupedDeviationsByRegion,    // Islay, Speyside etc.
    byCask: groupedDeviationsByCask,        // Sherry, Bourbon etc.
    byPeatLevel: groupedDeviationsByPeat,   // smoky vs non-smoky
    byAge: groupedDeviationsByAge,          // young vs aged
  },
  
  // 3. Blind vs. Open Tasting Vergleich
  blindVsOpen: {
    avgScoreBlind: avgScoreInBlindTastings,
    avgScoreOpen: avgScoreInOpenTastings,
    delta: avgScoreBlind - avgScoreOpen,
    // positiv = strenger wenn blind, negativ = großzügiger wenn blind
  },
  
  // 4. Konsistenz-Metrik
  consistency: {
    scoreStdDev: standardDeviationOfAllScores,
    label: stdDev < 8 ? 'consistent' : stdDev < 15 ? 'moderate' : 'exploratory'
  },
  
  // 5. Entwicklung über Zeit
  evolution: {
    firstQuarterAvg: avgScoreFirstQuarter,
    lastQuarterAvg: avgScoreLastQuarter,
    trend: lastQuarterAvg > firstQuarterAvg + 3 ? 'rising' : 
           lastQuarterAvg < firstQuarterAvg - 3 ? 'dropping' : 'stable'
  },
  
  // 6. Nase-First oder Finish-First Taster
  dominantDimension: dimensionWithHighestAvgRelativeToOverall,
}
```

### Neuer GPT-4o System Prompt

```
You are writing a personal tasting profile for a whisky enthusiast.

TONE: Warm, precise, epistemically humble. You are a skilled observer, 
not a diagnostician. You describe patterns in behaviour, not personality.
You write like a thoughtful sommelier who has been quietly watching 
someone taste for months.

LANGUAGE: Write in {{language}}. 
If German: use "du" (informal), warm but not casual.
If English: second person "you", elegant but not stiff.

STRUCTURE — three parts, flowing prose, no headers, no bullet points:

PART 1 — WHAT THE DATA SHOWS (60-80 words)
Describe 2-3 concrete, specific patterns from the analytics data.
Use precise numbers where they add insight. Be descriptive, not evaluative.
Example: "Your nose scores are consistently your most generous dimension — 
averaging 7 points above your own overall scores. Your finish scores, 
by contrast, are your most exacting."

PART 2 — WHAT THIS MIGHT MEAN (60-80 words)  
One or two careful interpretations. Use hedged language:
"This could suggest...", "One reading of this...", "It's possible that..."
Never state personality conclusions. Stay close to the behaviour.
Example: "This could suggest that first impressions matter most to you —
that you decide quickly and refine slowly."

PART 3 — ONE OPEN QUESTION (20-30 words)
End with a single genuine question that invites reflection.
Not rhetorical. Something the data raises but cannot answer.
Example: "Have you noticed that your scores in blind tastings 
run 8 points lower than when you know the bottle?"

TOTAL LENGTH: 150-200 words maximum. Less is more.
DO NOT: list features, use bullet points, mention the app by name,
make personality diagnoses, use superlatives.
```
