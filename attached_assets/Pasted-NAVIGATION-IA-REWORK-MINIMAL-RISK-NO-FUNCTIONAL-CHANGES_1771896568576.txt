NAVIGATION IA REWORK — MINIMAL RISK (NO FUNCTIONAL CHANGES, NO ROUTE CHANGES)

Context / Current pain:
Navigation is still too complex. We must reduce cognitive load for three core user situations:
1) Invited participant: only wants to join, rate, see recap, download export.
2) Host: prepare, run/administer, and evaluate tastings.
3) Enthusiast/Power user: deeper interest (journal, collection) + analytics + knowledge/research + fun features.
Important: The scientific/knowledge interface (lexicon, research, distilleries, bottlers, maps, database/library) must remain available.

GOAL:
Radically simplify the visible menu by reorganizing ONLY the navigation information architecture (groups, labels, ordering, collapsible subgroups), while keeping 100% of existing routes and features accessible.
This must feel premium, calm, and easy—especially for invited participants.

- DO NOT refactor business logic, tasting logic, contextLevel logic, or view logic.
- ONLY change navigation configuration + labels (i18n) + optional UI for collapsing groups.
- Keep icons as-is unless already broken.
- Compile clean, no runtime errors.
