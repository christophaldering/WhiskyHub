------------------------------------------------------------
MOBILE NAV KONZEPT (VERBINDLICH) – OPTION A
------------------------------------------------------------

1) Mobile hat genau EINE Hauptnavigation: Bottom Navigation mit 5 Tabs.
2) Mobile hat KEIN Hamburger-Menü und KEINE mobile Sidebar.
3) Unterseiten werden innerhalb der jeweiligen Tab-Seiten als Tabs/Segmented-Control angezeigt (wie Desktop-Zieldesign).
4) Alle bisherigen Features bleiben erreichbar:
   - Alles, was bisher im Hamburger/Sidebar erreichbar war, muss weiterhin erreichbar sein:
     -> auf Mobile über den Tab „Profil“ (Settings/More Bereich innerhalb Profil)
   - Admin ist nur für Admin sichtbar, aber erreichbar über Profil.

BOTTOM NAV (5 Tabs, fix):
1. Jetzt
2. Tastings
3. Tagebuch
4. Entdecken
5. Profil

MOBILE LABEL REGEL (WICHTIG):
- Labels müssen kurz sein und unter das Icon passen.
- Maximal 10 Zeichen (inkl. Leerzeichen), bevorzugt 4–8 Zeichen.
- Keine zweizeiligen Labels.
- Wenn zu lang: kürzen.

Vorgeschlagene Labels (bitte genau so verwenden, sofern möglich):
- Jetzt
- Tastings
- Tagebuch
- Entdecken
- Profil

Wenn „Tastings“ nicht passt: „Tasting“ (ohne s)
Wenn „Entdecken“ nicht passt: „Explore“

------------------------------------------------------------
DETAILS PRO TAB (MOBILE)
------------------------------------------------------------

TAB „Jetzt“:
- Route: /app (oder aktueller Lobby/Home-Route)
- Inhalt: Lobby / Continue / aktuelle Hinweise

TAB „Tastings“:
- Route: /sessions (oder bestehende Tastings-Übersicht)
- Untertabs IN DER SEITE (Segmented-Control): Sessions | Kalender | Host | Recap
- Keine extra Navigationsebene.

TAB „Tagebuch“:
- Route: /journal (oder bestehende Journal-Seite)
- Untertabs IN DER SEITE: Journal | Verkostet | Vergleich | Recap | Analytik (PRO) | Bibliothek (PRO) | Export

TAB „Entdecken“:
- Route: /recommendations oder /discover (bestehende Discover-Landing)
- Untertabs IN DER SEITE: Empfehlungen | Destillerien | Community

TAB „Profil“:
- Route: /profile
- Innerhalb Profil eine „Mehr/Settings“-Liste (keine Hamburger-Sidebar):
  - Konto (/account)
  - Hilfe (/help)
  - Über (/about)
  - Features (/features)
  - Spenden (/donate)
  - PRO Bereiche (Analytics/Data Export etc.) falls vorhanden
  - Admin (/admin) NUR für Admin sichtbar
  - Sprache / Dark Mode Settings bleiben erreichbar (UI-Controls können hier leben)

WICHTIG:
- Keine Funktionen entfernen. Nur die Einstiegspunkte werden auf Mobile in „Profil“ gebündelt.

------------------------------------------------------------
UMSETZUNG (TECHNISCH) – STRIKT UI/ROUTING UNVERÄNDERT
------------------------------------------------------------

1) Identifiziere die Komponente(n) für:
   - Desktop Sidebar Navigation
   - Mobile BottomNav
   - Mobile Hamburger Drawer / mobile Sidebar

2) Implementiere Responsive Layout-Switch:
   - Desktop (>= md): Sidebar bleibt wie bisher.
   - Mobile (< md): Sidebar/Hamburger wird NICHT gemountet/angezeigt.
     Stattdessen: BottomNav ist sichtbar.

3) Entferne auf Mobile:
   - Hamburger Button im Header (nur mobile)
   - Drawer/Sidebar overlay (nur mobile)
   Desktop unverändert.

4) Sicherstellen:
   - Jede Route bleibt wie bisher erreichbar (mindestens über Profil-Liste auf Mobile).
   - Keine Route wird “unlinked”.

5) Text-Längen prüfen:
   - BottomNav Labels dürfen nicht umbrechen.
   - Falls nötig: CSS/Classes anpassen (text-xs, leading-none, truncate).

6) Premium Ruhe:
   - BottomNav minimalistisch, hell, ohne harte Border.
   - Aktiver Tab: warmes Braun.
   - Inaktive Tabs: grau.
   - Keine starken Farben, keine dicken Linien.
