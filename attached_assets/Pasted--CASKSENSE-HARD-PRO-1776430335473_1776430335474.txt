SCHRITT 4 — DIAGNOSE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Basierend auf den Funden aus Schritten 1-3: Identifiziere die GENAUE Stelle
die dafür sorgt dass der Whiskys-Tab leer ist.

Mögliche Ursachen (prüfe welche zutrifft):

URSACHE A — Backend-Filter:
Die Query in routes.ts oder storage.ts filtert auf isDraft = false
oder ähnliches, was alle Draft-Einträge ausschließt.

URSACHE B — dataSnapshot-Problem:
Der Connoisseur Report speichert beim Generieren einen dataSnapshot in der DB.
Der Whiskys-Tab zeigt Daten aus diesem dataSnapshot — der zum Zeitpunkt
der Generierung vielleicht 0 Whiskys hatte weil der Filter zu streng war.

URSACHE C — Frontend-Filter:
Die Whiskys-Tab Komponente filtert die empfangenen Daten nochmals
nach isDraft oder status.

URSACHE D — Falsches Datenfeld:
Der Whiskys-Tab liest ein Feld (z.B. "whiskySummaries") das leer ist,
während die Daten in einem anderen Feld (z.B. "whiskyList") liegen.

SCHRITT 5 — FIX JE NACH URSACHE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Führe NUR den Fix für die in Schritt 4 identifizierte Ursache aus:

BEI URSACHE A (Backend-Filter):
  Finde die Query die isDraft = false oder ähnliches filtert.
  Ändere sie so dass Einträge mit vorhandenem Score eingeschlossen werden:

  Ersetze die isDraft-Bedingung durch:
    .where(
      or(
        isNotNull(journalEntries.overallScore),
        isNotNull(journalEntries.noseScore),
        gt(journalEntries.overallScore, 0)
      )
    )

  Falls der exakte Spaltenname anders heißt (aus Schritt 1/2 ermitteln):
  Passe den Spaltennamen entsprechend an.

BEI URSACHE B (dataSnapshot-Problem):
  Der gespeicherte Report hat einen leeren Snapshot. Es gibt zwei Optionen:

  Option B1 — Report neu generieren lassen:
    Füge im GET-Endpoint für den Connoisseur Report eine Live-Query hinzu
    die die Whiskys NICHT aus dem dataSnapshot liest, sondern direkt aus
    journal_entries für diesen Teilnehmer — mit dem Fix aus Ursache A.

  Option B2 — dataSnapshot beim Generieren korrigieren:
    Im POST-Endpoint (Generierung): Die Query die whiskySummaries befüllt
    muss isDraft-Filter entfernen und stattdessen nach Score filtern
    (wie in Ursache A beschrieben).

BEI URSACHE C (Frontend-Filter):
  In der Whiskys-Tab Komponente: Entferne den isDraft-Filter.
  Zeige alle Einträge die einen Score haben.

BEI URSACHE D (Falsches Datenfeld):
  Korrigiere den Whiskys-Tab so dass er das korrekte Datenfeld liest.
  Prüfe welches Feld tatsächlich Daten enthält und verwende dieses.
