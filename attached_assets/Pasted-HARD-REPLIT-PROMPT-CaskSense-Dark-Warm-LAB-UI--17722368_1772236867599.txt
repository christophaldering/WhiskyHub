HARD REPLIT PROMPT (CaskSense / Dark Warm LAB UI)
=================================================

ROLE
You are the lead engineer + UI architect for the existing CaskSense codebase (React/TS frontend, Express/TS backend, shared schema with Drizzle/Zod). Your task is to introduce a NEW parallel UI surface “Dark Warm Lab” WITHOUT changing the database schema and WITHOUT creating any new database, tables, or migrations. Both the existing UI and the new Lab UI MUST use the identical DB and the identical backend API routes. This is non-negotiable.

PRIMARY GOAL
Create a fully functional parallel UI at route prefix:
/lab-dark

It must include 3 pages:
1) /lab-dark/home
2) /lab-dark/sessions (list / overview)
3) /lab-dark/discover

AND a detail route for active tasting:
4) /lab-dark/session/:sessionId (or the project’s existing session id param style)

The Lab UI must feel “Apple-clean + whisky-warm (Dark Warm)” and be production-grade, but it is an experiment surface that must not disturb the current UI.

STRICT DB / API RULES
- DO NOT add or modify DB schemas.
- DO NOT create migrations.
- DO NOT introduce a second DB connection.
- DO NOT create “v2” API endpoints.
- DO NOT duplicate business logic on the server.
- ALL data used by /lab-dark must come from the existing API endpoints and existing shared hooks/services.
- If a required API does not exist, add it ONLY if it is general-purpose and ALSO used by the current UI, but do not change DB schema.

SAFETY REQUIREMENTS
- The existing UI routes must continue to work unchanged.
- The new /lab-dark routes must be fully isolated in UI code, but reuse the same data layer.
- Use feature toggles only inside the UI if needed; do not change the DB.
- If authentication/role guarding exists, follow the same guard rules for /lab-dark routes.
- Ensure navigation does not accidentally link regular users into /lab-dark unless explicitly intended (keep /lab-dark accessible via direct URL; optionally add a hidden “Lab” link only visible to admin/dev if such role exists).

   - active session / sessions list

- Ensure high contrast and accessible focus states.

Do NOT create a separate API client.

  useSessions(), useSession(sessionId), useBottles(), useTemplates(), usePairings(), useKnowledge()

  - Extract shared hooks from existing pages into reusable hooks in:

A) LAB HOME (/lab-dark/home)
- Header: “Good evening, Christoph” (if user name available; else “Good evening”)

   - If an active session exists: show session title, meta, primary CTA “Continue” links to /lab-dark/session/:id

   - Quick Log (bottom sheet)

D) LAB SESSION DETAIL (/lab-dark/session/:sessionId)
