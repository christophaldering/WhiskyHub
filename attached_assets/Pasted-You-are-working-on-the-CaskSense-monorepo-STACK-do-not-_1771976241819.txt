========================================================
A) BINDING LAYOUT CONTRACT (MUST FOLLOW)
========================================================

1) Every app page MUST be wrapped in <PageLayout>.
   NO page may implement its own top-level container, header, or navigation shell outside PageLayout.

2) PageLayout MUST support EXACTLY these official variants:
   - variant="default"  (standard content width)
   - variant="narrow"   (for form-heavy pages like Profile)
   - variant="immersive" (for special immersive pages such as Wishlist animations; later for Naked mode)
   Default is "default".

3) Container widths MUST be enforced centrally in PageLayout (not per-page):
   - default  => max-w-5xl mx-auto px-4 py-8
   - narrow   => max-w-2xl mx-auto px-4 py-8
   - immersive=> max-w-5xl mx-auto px-4 (reduced vertical padding allowed), still visually consistent

4) Page header MUST be rendered ONLY by PageLayout.
   PageLayout header supports:
   - icon + title (required)
   - optional subtitle
   - optional primary action (top right)
   - optional tabs row (below title)

5) Tabs rule (strict):
   - At most ONE top tabs row per page.
   - If a page has tabs, they MUST be rendered via PageLayout (not in page body).
   - Secondary filters are allowed only inside content sections, not as a second navigation row.

6) i18n rule (strict):
   - All PageLayout header labels (title, subtitle, tab labels, action labels) MUST come from i18n keys (react-i18next).
   - No hard-coded English strings in headers/tabs.

7) Exceptions rule:
   - /profile is NOT an exception: it MUST use PageLayout with variant="narrow".
   - /my/wishlist is NOT an exception: it MUST use PageLayout with variant="immersive" and keep AnimatePresence inside the page body.
