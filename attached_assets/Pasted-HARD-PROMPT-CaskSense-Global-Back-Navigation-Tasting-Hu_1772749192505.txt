PART 1 — GLOBAL NAVIGATION RULE: EVERY PAGE MUST HAVE "ZURÜCK"
------------------------------------------------

Goal:
Every page in the application must show a visible "← Zurück" control that returns to the previous in-app page.

Requirements:

1. Implement a global navigation history stack.

Use:
sessionStorage OR Zustand navigation store.

Store:
last 20 in-app routes.

On each route change:
push pathname + query to stack.

Ignore:
external URLs.

2. Back button logic priority:

1) query parameter "from"
2) navigation stack previous route
3) browser history.back()
4) fallback route

Fallback rules:
- if current route starts with /tasting → fallback /tasting
- if current route starts with /my-taste → fallback /my-taste

3. Implement a universal BackButton component.

Location suggestion:
client/src/components/navigation/BackButton.tsx

Properties:
- optional fallback
- optional label

Use i18n keys:
de: Zurück
en: Back

4. Ensure BackButton appears automatically in PageHeader.

Every page using PageHeader must show BackButton by default.

No page should omit BackButton.
