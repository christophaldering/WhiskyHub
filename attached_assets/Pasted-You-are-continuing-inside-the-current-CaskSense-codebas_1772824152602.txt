B. REQUIRED SCREENS
======================================================================

Build these screens fully:

1) Historical Tastings Archive Page
Purpose:
- overview of all historical tastings

Content:
- searchable list/grid of historical tastings
- each card/row shows:
  - tasting number
  - title
  - optional date
  - number of whiskies
  - winner / top whisky if available
  - average total score
- filter/sort controls:
  - sort by tasting number
  - sort by newest/oldest if dates exist
  - sort by average quality
  - optionally filter by smoky / region / cask if data supports this

2) Historical Tasting Detail Page
Purpose:
- inspect one tasting in depth

Content:
- tasting header
- tasting metadata
- ranked lineup table/cards
- visual emphasis on winner/top 3
- per whisky:
  - distillery
  - name
  - age
  - abv
  - cask
  - nose/taste/finish/total
  - ranks
  - smoky/ppm if available
- tasteful mini visualizations where useful
- excellent empty/fallback handling if some fields are missing

3) Historical Insights / Analytics Page
Purpose:
- turn archive into intelligence

Content:
- top-rated whiskies across all historical tastings
- most frequently tasted whiskies
- best-performing regions
- smoky vs non-smoky comparison
- cask-type comparison
- price vs score exploration if feasible with available data
- “group taste profile” section:
  - what this tasting group seems to like
  - sherry tendency
  - peat tendency
  - ABV tendency
  - region tendency

4) Whisky Detail Integration
If the app already has whisky detail/profile surfaces, integrate historical insights there:
- appeared in X historical tastings
- avg total score
- best historical placement
- last historical appearance
- link to related tastings
