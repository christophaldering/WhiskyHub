HARD PROMPT (Replit) — Session sauber lösen: sichtbarer Sign-in/Sign-out Entry + einheitliche Auth für Classic & Simple

GOAL:
Make session/account handling obvious, consistent, and accessible in BOTH Classic View and Simple Mode:
- Always provide a visible entry point to session (profile icon)
- Provide Sign in / Sign out in a single “Session” bottom sheet
- Use the SAME auth state everywhere (Classic pages + /log-simple + tasting)
- Remove “hidden logout” reliance on the ⋯ menu

NO DB migrations. No real account system required. Keep lightweight PIN-based “sign in”.

────────────────────────────────────────────────────────
A) SINGLE SOURCE OF TRUTH (AUTH STATE)
Create/confirm a single auth module used by all UIs:

Frontend module: client/src/lib/session.ts (or reuse simple-auth but rename/export as shared)
Must expose:
- getSession(): { signedIn:boolean, name?:string|null, mode?:"log"|"tasting", remember?:boolean }
- signIn({ pin, name?, mode, remember }): Promise<void>
- tryAutoResume(): Promise<void>  // uses resumeToken if present
- signOut(): Promise<void>

Storage rules:
- sessionStorage:
  - session_signed_in="1"
  - session_mode="log"|"tasting"
  - session_name="<string>" (log only; optional)
- localStorage (only if remember enabled):
  - session_remember="1"
  - session_resume_token="<token>"
  - session_resume_mode="log"|"tasting"
  - session_resume_name="<string>" (log only optional)

Never store plaintext PIN.

Auto-resume:
- Call tryAutoResume() once at app start (top-level App mount or Layout mount),
  before rendering headers so “Good morning, {name}” is correct.

────────────────────────────────────────────────────────
B) BACKEND (TOKEN-BASED RESUME)
Ensure these endpoints exist and are used by BOTH Classic & Simple:

1) POST /api/session/signin
Body: { pin:string, name?:string, mode:"log"|"tasting", remember:boolean }
- Validate pin via env SIMPLE_MODE_PIN (required)
- If mode==="log": require name only if env SESSION_LOG_REQUIRES_NAME==="true"
- If mode==="tasting": ignore name (PIN-only)
- If remember===true: return { ok:true, resumeToken }
- Else return { ok:true }

2) POST /api/session/resume
Body: { resumeToken:string }
- If valid: return { ok:true, mode, name? }
- Else 401

3) POST /api/session/signout
Body: { resumeToken?:string }
- Invalidate token if provided
- Return ok:true

No DB required:
- In-memory token map with TTL 14 days is fine for MVP.
- If server restarts, resume may fail → client clears stored token and shows signed out.

Rate limiting:
- signin attempts: 5/5min per IP
- resume: 30/min per IP
